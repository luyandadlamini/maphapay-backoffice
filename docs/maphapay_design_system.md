# MAPHAPAY DESIGN SYSTEM
Version 1.0

---

## 1. PURPOSE

This design system ensures MaphaPay feels:
- Human
- Effortless
- Trustworthy
- Modern and culturally relevant to Eswatini

In fintech, design is not decoration — it is how trust is built and maintained. Poor UX leads directly to user drop-off and loss of confidence. :contentReference[oaicite:0]{index=0}

---

## 2. CORE DESIGN PRINCIPLES

### 1. Human > Interface
Every screen must reflect real life.
If there is no human context, the design is incomplete.

---

### 2. Clarity over cleverness
Users should never think about how to use the app.

- Reduce steps
- Reduce text
- Reduce decisions

---

### 3. Show, don’t explain
Use visuals instead of descriptions:
- Phone mockups
- Real people
- Real scenarios

---

### 4. Trust is designed, not claimed
Trust comes from:
- Clear feedback
- Predictable behaviour
- Transparency

---

### 5. Emotion before product
Users should feel:
"This is for me"
before they understand features

---

## 3. LAYOUT SYSTEM

### Grid

- Desktop: 12-column grid
- Tablet: 8-column grid
- Mobile: 4-column grid

---

### Spacing Scale

- XS: 8px  
- SM: 16px  
- MD: 24px  
- LG: 40px  
- XL: 64px  
- XXL: 120px  

Premium design relies on spacing to reduce cognitive load.

---

### Section Rules

Each section must:
- Communicate one idea
- Have one visual focus
- Have one primary action

---

## 4. COLOUR SYSTEM

### Core Palette

- Blue → trust, structure
- Gold → action, energy
- Black → premium tone
- White → clarity and space

---

### Usage Rules

- Gold is ONLY for CTAs and highlights
- Blue should not dominate visuals
- Avoid using all colours together

---

## 5. TYPOGRAPHY SYSTEM

### Headings

- Bold
- Short (max 6 words)
- High contrast

---

### Body Text

- Simple, conversational
- Short sentences
- No jargon

---

### Copy Rules

Avoid:
- Technical language
- Long paragraphs
- Corporate tone

---

## 6. IMAGERY SYSTEM

### Categories

1. Friends (social payments)
2. Family (parent + child)
3. Everyday commerce (shops, taxis)
4. Individual empowerment

---

### Style

- Natural lighting
- Real environments
- Slight motion (not static)

---

### Reject If:

- Looks like stock photography
- Feels staged
- Not culturally relevant

---

## 7. COMPONENT SYSTEM

### Buttons

- Pill-shaped
- Large tap targets
- Gold = primary action

---

### Cards

- Rounded corners
- Soft shadows
- Minimal content

---

### Phone Mockups

- Real UI only
- Slight tilt (3D feel)
- Used to demonstrate actions

---

## 8. MOTION SYSTEM

### Principles

- Motion should guide attention
- Not distract

---

### Use

- Scroll reveals
- Soft transitions
- UI animations

---

### Avoid

- Fast flashy animations
- Overuse of motion

---

## 9. INTERACTION DESIGN

Key actions must be:
- Visible
- Immediate
- One-step where possible

Users expect instant actions in fintech — delays reduce trust.

---

## 10. TRUST DESIGN SYSTEM

Always include:

- Confirmation states
- Status updates
- Clear feedback

Users must always know:
- What is happening
- What just happened
- What happens next

---

## 11. CONTENT SYSTEM

### Tone

- Friendly
- Confident
- Local

---

### Writing Style

- Everyday language
- Action-based
- Short

---

### Example

❌ “Execute a transaction”  
✅ “Send money”

---

## 12. ANTI-PATTERNS

Do NOT:

- Overload features
- Use too much text
- Use generic imagery
- Create complex navigation
- Overuse colour

---

## 13. FINAL CHECK

Before shipping any design:

- Does it feel human?
- Does it feel simple?
- Does it build trust?
- Would someone feel proud using this?
